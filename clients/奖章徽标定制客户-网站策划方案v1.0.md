## 一、项目背景与理解

### 1.1 对客户需求的理解

基于与贵司的初步沟通，我们理解到：

- **产品**：金属奖章、徽标、胸针、挂件（胸前佩戴型）
- **目标市场**：海外（B2B），具体待确认
- **核心诉求**：对方客户"不太懂也不知道怎么设计"，需要一个能清晰展示定制能力、建立信任、促进询价的英文官网
- **客户状态**：潜在客户，有定制意向，需要引导和信任建立

### 1.2 网站的核心任务

这个网站要解决的最核心问题是：**"我怎么知道这家公司能不能做我的产品？"**

所有设计决策都围绕这个问题展开：

```
访客到达
    ↓
3秒内：我知道这家公司是做什么的 ✓
    ↓
10秒内：他们能做我要的产品类型吗？→ 产品分类清晰可见
    ↓
30秒内：这家公司可信吗？→ 案例展示 + 工厂实力 + 资质
    ↓
2分钟内：我该怎么询价？→ 明显的CTA + 简洁表单
    ↓
提交询价 → 24小时内收到回复
```

## 三、目标用户画像

### 画像A：海外体育赛事组织方（最核心目标用户）

**场景**：需要为即将举办的马拉松赛事定制 5,000 枚完赛奖牌。

**心理活动**：
- "奖牌要有质感，运动员才会在意"
- "我不懂工艺，但要找到看起来靠谱的供应商"
- "预算有限，但不能让奖牌看起来廉价"
- "交期紧，晚了就废了"

**决策路径**：
Google 搜索 → "custom marathon medal manufacturer" → 前3个结果 → 打开网站 → 找产品图 → 找最小起订量 → 找联系方式 → 发邮件询价

**网站需要做的**：让用户在30秒内找到"他们能做马拉松奖牌"的信心，并提供最简单的询价路径。

### 画像B：企业HR / 品牌经理

**场景**：公司10周年，需要定制 200 枚员工荣誉徽章，作为周年庆典礼品。

**心理活动**：
- "只有200枚，小批量能接吗？"
- "我不懂设计，只有品牌Logo"
- "能不能先看看样品再做决定？"

**网站需要做的**：重点展示"小批量灵活定制"能力，提供"没有设计图也可以询价"的安心承诺。

### 画像C：校友会/慈善机构负责人

**场景**：大学校友会50周年，需要定制 100 枚纪念章。

**心理活动**：
- "预算有限（每枚几十块钱人民币）"
- "只要设计好看就行，不需要太复杂"
- "希望流程简单，不要折腾"

**网站需要做的**：价格友好感展示（非低价低质，而是"小批量高性价比"），流程简单明了。

## 四、网站功能规划

### 4.1 必须有的功能（核心转化）

| 功能 | 作用 | 设计要点 |
|---|---|---|
| **产品分类展示** | 让用户快速判断"他们能做我的产品" | 5-6个分类卡片，配图标+简述 |
| **定制流程可视化** | 消除陌生感，让"不懂设计"的客户放心 | 时间轴，6步，图文并茂 |
| **询价表单** | 核心转化 | 2步引导，不超过6个必填字段 |
| **作品案例 Gallery** | 建立信任，证明能力 | 按行业分类，真实案例 |
| **工厂/关于我们** | 建立信任，证明实力 | 工厂实拍 + 数字数据 + 资质 |
| **工艺说明** | 差异化，展示专业深度 | 珐琅/压铸/电镀等工艺图解 |

### 4.2 应该有的功能（增强信任）

| 功能 | 作用 |
|---|---|
| 快速询价浮窗（底部固定） | 移动端随时可询价 |
| WhatsApp / WeChat 浮窗 | 即时通讯，降低联系门槛 |
| 客户 Logo 墙 | 知名客户背书 |
| 在线客服（Tawk.to） | 零成本接入 |
| 多语言切换（EN/ES/FR） | 覆盖更多海外市场 |

### 4.3 暂不需要的功能

- 电商/在线支付（B2B 询价模式不需要）
- 会员登录系统（增加维护成本）
- 博客系统（先做核心功能，上线后再考虑）

## 八、技术方案

### 方案一：WordPress（推荐）

**优势**：
- 客户后期可自行更新内容（添加案例/更新产品）
- 全球最广泛使用的 CMS，维护成本低
- 丰富的插件生态（表单/SEO/多语言/安全）
- 开发速度快，成本相对可控

**推荐插件组合**：

| 插件 | 用途 |
|---|---|
| Elementor Pro | 页面构建，拖拽可视化 |
| FluentForm | 询价表单，邮件通知 |
| Yoast SEO | 技术SEO配置 |
| WPML 或 Polylang | 多语言支持 |
| UpdraftPlus | 自动备份 |
| Wordfence | 安全防护 |
| ShortPixel | 图片压缩优化 |
| Tawk.to | 在线客服浮窗 |
| Complianz | GDPR/隐私政策合规 |

### 方案二：Webflow（备选）

**适用场景**：客户对设计要求极高，WordPress 默认主题无法满足需求。

**优势**：设计自由度更高，动画效果更强
**劣势**：客户无法自行更新内容（需每次付费找我们更新）；费用略高

### 方案三：静态站点（经济型）

**适用场景**：预算极为有限，只需要展示型网站，不需要后台。

**技术**：Astro + Tailwind CSS + Netlify（免费托管）
**优势**：极快加载速度，极低维护成本
**劣势**：内容更新需要修改代码

### 为什么要建一个专业网站？

**客户的潜在疑虑**："我已经有阿里巴巴店铺了，还需要单独建网站吗？"

**回答**：

- **品牌差异**：阿里巴巴店铺的视觉和询价流程是被平台固定的，买家同时看到几十家供应商，没有差异化。一个独立网站让你有完整的品牌叙事权。
- **信任建立**：有独立官网的企业在海外买家眼中"看起来更正规"。很多采购负责人会先 Google 你的公司，官网是他们评估你的第一步。
- **SEO 流量**：通过 Google 搜索找到你的客户，比平台询价的客户转化率高 3-5 倍（因为他们主动搜索，意向明确）。
- **数据资产**：询价客户的联系方式在你自己的数据库里，不在平台上。

### 一个好网站的标准是什么？

**给客户看的 Checklist：**

```
□ 首页能让我在10秒内知道这家公司是做什么的
□ 产品分类清晰，我知道去哪找我要的产品
□ 定制流程有说明，我大概知道要多久、怎么走
□ 有真实案例，不是随便放几张图片
□ 询价很方便，不需要先注册账号
□ 联系他们的方式很多（表单/邮件/WhatsApp）
□ 网站在手机上看起来也很好
□ 网站打开速度快（3秒以内）
□ 有英文版本（对于面向海外客户的我来说）
□ 有 HTTPS 安全标志（浏览器地址栏的小锁）
```

### 常见误区

**误区1："我先随便建一个，后面再改"**
→ 网站是品牌的门面，第一印象很难改。建议一次做好。

**误区2："把阿里巴巴的图和产品描述直接复制过去就行"**
→ 独立网站的价值在于差异化叙事，复制过去反而没有体现定制能力。

**误区3："网站建好了就不用管了"**
→ 网站需要持续更新案例和内容，才能在 Google 上保持排名。建议每月更新1-2个新案例。

**误区4："我不懂设计，让我做决策很难"**
→ 这正是我们的价值所在——我们会提供推荐方案和参考案例，您只需要选择方向，专业的事交给我们。
